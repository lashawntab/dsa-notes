Common Expected Output Characteristics

| **Output Aspect** | **What It Specifies**               | **Why It Matters**             |
| ----------------- | ----------------------------------- | ------------------------------ |
| Data Type         | int, bool, string, list, tuple      | Dictates operations and memory |
| Structure         | Single value, array, matrix, tree   | Determines construction cost   |
| Ordering          | Sorted, original order, any order   | Affects algorithm choice       |
| Uniqueness        | Unique values or duplicates allowed | Impacts use of sets            |
| Index vs Value    | Position, value, or both            | Changes what you track         |
| Count vs List     | Number of results or all results    | Impacts time and space         |
| Multiple Answers  | One valid or all valid              | Pruning vs enumeration         |
| Precision         | Exact, rounded, tolerance           | Float handling                 |
| Modulo            | Result mod N                        | Forces modular arithmetic      |
| Empty Case        | What to return if none found        | Edge-case handling             |
| Error Signaling   | Null, -1, exception                 | Contract compliance            |
| Stability         | Preserve relative order             | Sorting constraints            |
| Format            | JSON, plain list, custom object     | Output serialization           |
